    Open-core package split readiness for base/, services/, lifecycle/, plugins/

    Status: COMPLETE — all B1-B6 + E-L1/L2 + E-B1/B2/B3 + E-S2 + E-P1/P2/P3 shipped on evolution branch · Owner: platform · Created: 2026-06-25

    Context

    The Athomic framework is being prepared for an open-core model: a free OSS core (nala-athomic-core) plus paid proprietary extension packages distributed as standard Python packages that plug in via entry points (group="nala.plugins") and hooks (@hookimpl). A working proof-of-concept already exists in plugins/nala-plugin-greetings/.

    base/, services/, lifecycle/, and plugins/ are the four modules a plugin author will subclass/consume most — BaseRegistry (~85 subclasses), BaseService (~50 subclasses, 140+ import sites), and the plugin hook protocol. Before any real package split they needed a specific set of fixes that would break the multi-package model if left in place — not generic tech debt.

    Cross-references to the larger platform maturity backlog: - lifecycle_plugins.md — 23 EPICs (P0-P5) for full lifecycle/plugin observability, restart policies, grafo de dependência, etc. Nothing there is a prerequisite for the first paid plugin. - factories-pattern-standardization.md — Factory pattern audit. - circular-imports-violations.md — import architecture debt.

    Tasks completed (B-series = split-blockers, E-series = same-file enhancements)

    B1 — PluginManager settings injection

    plugins/manager.py hard-called get_settings().plugins with no override. Added settings: Optional[PluginsSettings] = None constructor arg (falls back to global). facade.py now injects settings=self.settings.plugins explicitly.

    B2 — De-singletonize lifecycle_registry + descentralize domain registration

    Two combined changes: 1. LifecycleManager(registry=…) — optional injection, default = global singleton. Athomic creates and owns its LifecycleRegistry instance. 2. Eleven new <domain>/lifecycle.py files replace the monolithic register_services.py import fan-out. register_services.py became a thin orchestrator that calls register_lifecycle() per domain.

    With E-L1: LifecycleManager.describe() for cheap startup diagnostics.

    With E-L2: DomainInitializerRegistry._initializers moved from a class attribute to an instance attribute (guarded by hasattr in __init__, since __init__ runs on every construction even for the singleton). Added clear() mirroring LifecycleRegistry.clear() for test isolation. Unit coverage added in tests/unit/nala/athomic/lifecycle/test_lifecycle_initializers.py.

    B3 — Add lifecycle/notification/services/base to import-linter layers + decouple hard imports

    • Added four new layers to athomic-layers contract; pre-existing cross-imports in the base/services/observability mutual cluster listed in ignore_imports so all 4 contracts keep passing.
    • plugins/manager.py: NotificationFactory module-level import replaced by Optional[NotificationProtocol] constructor injection + lazy import fallback.
    • lifecycle/manager.py: CacheHandler module-level import removed; new cache_handler_cls: Optional[type] constructor arg; facade.py injects CacheHandler at construction.

    Phase 0 base/ hardening (E-B1, E-B2, E-B3 + collateral fixes)

    Shipped together in commit 6653294 as quality/DX fixes independent of the split-blocker sequence:

    E-B1 — BaseManager.factory as explicit constructor parameter Previously a bare post-hoc attribute each subclass set in its own __init__ with no base-class knowledge. Now an optional factory keyword arg on BaseManager.__init__(), mirroring the existing resilience_service injection pattern. The NotImplementedError guard in before_start() was updated to point to the constructor, not the subclass. Subclasses that fully override before_start() (e.g. RegularConsumerOrchestrator, RepublisherManager) never needed a factory and are unaffected.

    E-B2 — RegistryItemNotFoundError with .available_keys BaseRegistry.get() and BaseInstanceRegistry.get() both raised a generic ValueError on a miss. Replaced with RegistryItemNotFoundError(AthomicError, ValueError) carrying .key, .available_keys, and .registry_name. The ValueError base is kept for backward compatibility — existing callers doing except ValueError around registry lookups keep working unchanged.

    E-B3 — Remove dead load_external_plugins() / PluginBroadcasterProtocol BaseRegistry.load_external_plugins() and its PluginBroadcasterProtocol were never called anywhere in the codebase (duplicated PluginManager's own discovery channel). Removed entirely. Collateral in the same commit: - Deduplicated the @runtime_checkable Protocol detection logic from BaseRegistry and BaseInstanceRegistry into a single is_runtime_checkable_protocol() helper in registry.py. - Fixed a concurrency bug in BaseInstanceRegistry.clear() — previously could silently drop and never tear down an instance registered concurrently with an in-flight clear(); fixed by atomically swapping the dict under the lock. - Fixed BaseManager.health() returning ready=True for a manager with zero configured clients (all() over an empty iterable is vacuously True). - Fixed _resolve_active_settingss typo (now _resolve_active_settings).

    B5 — ServiceState transition validation

    services/enums.py: VALID_TRANSITIONS table. services/base.py: _set_state() enforcer — raise on invalid transition (not warn-only). ServiceStateTransitionError added to services/exceptions.py. With E-S2: BaseService.__repr__ showing class name + service name + state.

    B6 — Validate hooks against AthomicHooks.KNOWN_HOOKS

    _register_hooks_from_module now rejects unknown hook names with a warning instead of silently registering them. With E-P1: hookimpl supports optional @hookimpl(priority=N) — lower runs first; call_hook sorts by _hook_priority. With E-P2: PluginManager.describe_plugin(name) — returns {name, hooks}. With E-P3: PluginFatalError (plugins/exceptions.py) — re-raised by call_hook so a plugin can abort the boot sequence.

    Acceptance criteria (all verified on evolution branch)

    • make solid — 4 contracts kept, 0 broken; 0 new TYPE_CHECKING violations.
    • make test — 3104 passed (was 3085 pre-session); 3 pre-existing outbox errors require live infra and are excluded via --ignore.
    • Smoke: Athomic(lifecycle_registry_override=LifecycleRegistry(...)) runs isolated from the global singleton.

    Not in this backlog (explicitly deferred)

    Everything in lifecycle_plugins.md (EPICs 1-23) remains valid future work and is not duplicated here. Key deferred items: - Full singleton elimination beyond lifecycle_registry (EPIC-01). - Health/ready split, restart policies, TaskGroupManager (EPICs 02-03). - Plugin manifest + semantic version compat range (EPIC-13) — only needed when multiple independent third-party plugins exist in production. - Physical package extraction (nala-athomic-core as a separate PyPI distribution, EPIC-19) — orthogonal decision, can be made after B1-B6 are stable. - E-S1 (BaseService metrics/tracer injection) — requires a new ServiceMetricsRecorder Protocol; deferred as out-of-scope for this pass.
