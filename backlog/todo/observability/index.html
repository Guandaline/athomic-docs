    Observability 2.0 Backlog (Athomic Layer)

    EPIC 1 — ObservabilityCore (Unified Observability Layer)

    Goal

    Create a single module that centralizes logging, metrics, tracing, instrumentations and execution context.

    Tasks

    • [ ] Create module nala/athomic/observability/core.py
    • [ ] Register ObservabilityCore as a first-class Athomic service
    • [ ] Centralize initialization of:
    • [ ] logging
    • [ ] tracing
    • [ ] metrics
    • [ ] OTel auto/semiauto instrumentation
    • [ ] execution context propagation (tenant_id, trace_id)
    • [ ] Create ObservabilitySettings
    • [ ] Ensure all services enrich logs + metrics with contextual fields
    • [ ] Provide unified API:
    • python observability_core.logger observability_core.metrics observability_core.tracer

    EPIC 2 — Metrics Style Guide (Standardization)

    Goal

    Define strict conventions for naming, labeling and structuring all metrics.

    Tasks

    • [ ] Create file: docs/observability/metrics-style-guide.md
    • [ ] Define global naming convention:
    • domain_action_metricname
    • [ ] Define mandatory labels:
    • tenant_id, service, provider, status
    • [ ] Define standard histogram buckets
    • [ ] Create templates for:
    • counters
    • histograms
    • gauges
    • [ ] Implement @instrumented decorator
    • [ ] Write metric compliance tests
    • [ ] Apply standard metrics to all Athomic modules

    EPIC 3 — Distributed Tracing Deep Integration

    Goal

    Enable full end-to-end tracing coverage across messaging, payload pipeline, KVStore, HTTP clients and core services.

    Tasks

    Messaging Layer

    • [ ] Add spans for:
    • message processing
    • batch processing
    • orchestration strategy
    • execution strategy
    • retry / delay logic
    • DLQ forwarding
    • [ ] Include attributes:
    • message_id, partition, offset, batch_size

    Payload Pipeline

    • [ ] Add spans for each pipeline step:
    • serialize
    • compress
    • encrypt
    • [ ] Register exceptions as span events
    • [ ] Add attributes:
    • original_size
    • final_size
    • algorithm
    • step_latency

    KVStore

    • [ ] Add spans for:
    • get, set, delete, exists, expire
    • [ ] Add attributes:
    • key_type, size, ttl, provider_name

    HTTP Client

    • [ ] Wrap httpx requests with spans
    • [ ] Add retry information into spans
    • [ ] Add attributes:
    • status_code, url_base, latency_ms
