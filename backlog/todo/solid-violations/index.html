    SOLID Violations Backlog

    All violations of the rules defined in docs/guides/architecture/solid-rules.md. Every item here must be fixed. No new TYPE_CHECKING imports may be added.

    Priority Levels

    • P0 — Critical: Cross-layer DIP violations (high layer imported by low layer). Fix immediately.
    • P1 — High: Within-package circular imports papered over with TYPE_CHECKING. Fix before next release.
    • P2 — Acceptable: Optional external dependencies (try/except import pattern). Not a violation.

    Fix Grouping Strategy

    When fixing P1 violations, group by package to avoid multiple PRs touching the same files:

    Group Violations Fix approach
    G1 — resilience/adaptive_concurrency V-004–V-008 resilience/adaptive_concurrency/protocols.py + feedback/types.py
    G2 — integration/producers V-009–V-011 integration/messaging/producers/types.py
    G3 — integration/consumers V-012–V-020 integration/messaging/consumers/types.py + messaging/protocols.py
    G4 — integration/orchestration V-021–V-023 integration/messaging/causality/types.py
    G5 — plugins/facade V-002, V-003 nala/athomic/interfaces/athomic.py
    G6 — database/resilience V-024–V-026 Local Protocols in database/migrations/ + database/kvstore/
    G7 — observability domain probes V-030–V-034 Move probes UP to domain layer; use local Protocols
    G8 — ai/features forward refs V-027–V-029 from __future__ import annotations + tensors/types.py
    G9 — config/schemas/control V-035 Move factory call to facade; schema stays pure
