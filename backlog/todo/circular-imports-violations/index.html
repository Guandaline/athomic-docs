    Circular Import Violations Backlog

    Priority: Critical Architecture Debt. All 18 files in KNOWN_OPEN (scripts/no_type_checking.py) must be fixed. As each file is fixed, remove it from KNOWN_OPEN. When KNOWN_OPEN is empty, delete the whitelist block from the script.

    See docs/guides/architecture/circular-imports-rules.md for the decision tree and the 3 valid fix patterns.

    Classification Summary

    Category Count Files
    True circular import — needs Protocol extraction 5 producers ×3, consumers base ↔ consumption protocol
    True circular import — needs neutral data module 1 algorithms/telemetry

    Group 1: Not Circular — Regular Imports Suffice (no design change needed)

    These files use TYPE_CHECKING / from __future__ import annotations purely out of habit or excessive caution. The imported type does not create any cycle. Fix is trivial: remove the guard and move to a plain import.

    VIO-01 consumers/strategies/orchestration/protocol.py

    Root cause: VectorClock is imported under TYPE_CHECKING as a parameter type. VectorClock is in nala.athomic.causality.vector_clock — a lower-level module that does not import anything from messaging. No cycle exists.

    Fix:

    # Remove:
from typing import TYPE_CHECKING, ...  # remove TYPE_CHECKING
if TYPE_CHECKING:
    from nala.athomic.causality.vector_clock import VectorClock

# Add:
from typing import ...                  # drop TYPE_CHECKING
from nala.athomic.causality.vector_clock import VectorClock

# Change in method signature:
incoming_vc: Optional["VectorClock"] = None
# becomes:
incoming_vc: Optional[VectorClock] = None

    Files to change: consumers/strategies/orchestration/protocol.py

    VIO-02 consumers/strategies/orchestration/providers/direct.py

    Root cause: Same as VIO-01. VectorClock used in process() signature.

    Fix: Identical to VIO-01.

    Files to change: consumers/strategies/orchestration/providers/direct.py

    VIO-03 consumers/strategies/orchestration/providers/ordered.py

    Root cause: Same as VIO-01. VectorClock used in process() signature (marked incoming_vc is explicitly ignored by this strategy).

    Fix: Identical to VIO-01.

    Files to change: consumers/strategies/orchestration/providers/ordered.py

    VIO-04 consumers/strategies/creators.py

    Root cause: ConsumptionStrategyProtocol (from consumption/protocol.py) is imported under TYPE_CHECKING. The protocol lives below creators.py in the import chain — it does not import creators.py. No cycle.

    Fix: Move ConsumptionStrategyProtocol to a regular import. Remove TYPE_CHECKING from the typing import if it's no longer used.

    Files to change: consumers/strategies/creators.py

    VIO-05 providers/local/strategies/creators.py

    Root cause: ConsumptionStrategyProtocol is under TYPE_CHECKING. It imports from consumers/strategies/consumption/protocol.py — which does not import back into providers/local/. No cycle. The guard was added defensively.

    Fix: Move ConsumptionStrategyProtocol to a regular import.

    Files to change: providers/local/strategies/creators.py

    VIO-06 consumers/processors/creators.py

    Root cause: MessagingSettings, ConnectionManager, ConsumerHandlerConfig, DLQHandler, SerializerProtocol are all under TYPE_CHECKING. None of these modules import from processors/creators.py. No cycle.

    Fix: Move all five to regular imports. Remove TYPE_CHECKING and from __future__ import annotations.

    Files to change: consumers/processors/creators.py

    VIO-07 providers/local/strategies/single.py

    Root cause: RawMessage is already a regular import. BaseConsumer and ProcessorProtocol are under TYPE_CHECKING. The cycle is NOT in this file — it is in consumption/protocol.py (see VIO-09). Once MessageSourceProtocol is introduced (VIO-09), this file switches its BaseConsumer annotation to MessageSourceProtocol, which has no cycle. Until VIO-09 is fixed, the TYPE_CHECKING guard holds the debt.

    Dependency: Fix VIO-09 first.

    Fix (after VIO-09):

    # Remove TYPE_CHECKING guard around BaseConsumer
# Use MessageSourceProtocol instead of BaseConsumer in method signatures
from nala.athomic.integration.messaging.consumers.strategies.consumption.types import (
    MessageSourceProtocol,
)

    Files to change: providers/local/strategies/single.py

    Same as VIO-07. Same dependency on VIO-09.

    Files to change: providers/local/strategies/batch.py

    VIO-09-kafka providers/kafka/strategies/single.py

    Same as VIO-07. Same dependency on VIO-09.

    Files to change: providers/kafka/strategies/single.py

    Group 2: True Circular Imports — Protocol Extraction Required

    VIO-09 consumers/base.py ↔ consumers/strategies/consumption/protocol.py

    The cycle:

    consumers/base.py
  └── [TYPE_CHECKING] imports ConsumptionStrategyProtocol from consumption/protocol.py

consumption/protocol.py
  └── [TYPE_CHECKING] imports BaseConsumer from consumers/base.py
      (used as parameter type in: consume(consumer: "BaseConsumer", processor: ...))

    Root cause: The consumption strategy needs to call consumer.get_message_iterator() and consumer.get_batch(). These are the only two methods strategies actually invoke on the consumer. Instead of referencing the concrete BaseConsumer, the protocol should depend on a minimal abstraction.

    Fix — extract MessageSourceProtocol:

    New file: src/nala/athomic/integration/messaging/consumers/strategies/consumption/types.py

    from typing import AsyncGenerator, List, Protocol

from nala.athomic.integration.messaging.raw_message import RawMessage


class MessageSourceProtocol(Protocol):
    async def get_message_iterator(self) -> AsyncGenerator[RawMessage, None]: ...
    async def get_batch(self) -> List[RawMessage]: ...

    Changes: - consumption/protocol.py: remove TYPE_CHECKING guard around BaseConsumer; use MessageSourceProtocol as the parameter type in consume(). - consumers/base.py: remove TYPE_CHECKING guard around ConsumptionStrategyProtocol; move to regular import (no cycle after protocol.py stops importing base.py). - BaseConsumer satisfies MessageSourceProtocol structurally — no implements needed. - VIO-07, VIO-08, VIO-09-kafka: update BaseConsumer annotation to MessageSourceProtocol.

    Files to create: consumers/strategies/consumption/types.py Files to change: consumption/protocol.py, consumers/base.py, local/single.py, local/batch.py, kafka/single.py

    VIO-10 producers/strategies/protocol.py ↔ producers/base.py

    The cycle:

    producers/base.py
  └── imports PublishingStrategyProtocol from strategies/protocol.py

strategies/protocol.py
  └── [TYPE_CHECKING] imports BaseProducer from producers/base.py
      (used in: publish(producer: "BaseProducer", ...) method signature)

    Root cause: Strategies access producer.backend, producer.tracer, producer.payload_processor, producer.key_header_serializer, producer._telemetry, producer._publish(). This is Feature Envy — strategies are reaching into the producer's internals. The fix surfaces those as a proper transport abstraction.

    Fix — extract ProducerTransportProtocol:

    New file: src/nala/athomic/integration/messaging/producers/transport.py

    from typing import Any, Optional, Protocol

from nala.athomic.integration.messaging.headers import MessageHeaders
from nala.athomic.integration.messaging.outcomes import PublishingOutcome
from nala.athomic.observability.tracing import TracerProtocol  # or appropriate type


class ProducerTransportProtocol(Protocol):
    backend: Any                          # the underlying Kafka/local backend
    service_name: str
    tracer: TracerProtocol
    payload_processor: Any                # PayloadProcessor
    key_header_serializer: Any            # KeyHeaderSerializer
    telemetry_adapter: Any                # ProducerTelemetryAdapter (or whatever name)

    async def dispatch_bytes(
        self,
        topic: str,
        payload: bytes,
        key: Optional[bytes],
        headers: MessageHeaders,
    ) -> PublishingOutcome: ...

    Changes: - BaseProducer must expose telemetry_adapter and dispatch_bytes() as public members (rename the private _telemetry → telemetry_adapter; expose _publish() via dispatch_bytes()). - strategies/protocol.py: remove TYPE_CHECKING guard; use ProducerTransportProtocol as the parameter type in publish(). - strategies/direct.py, strategies/delayed.py: remove TYPE_CHECKING guard around BaseProducer; use ProducerTransportProtocol in all method signatures. - BaseProducer satisfies ProducerTransportProtocol structurally.

    Files to create: producers/transport.py Files to change: producers/base.py (expose telemetry_adapter + dispatch_bytes), strategies/protocol.py, strategies/direct.py, strategies/delayed.py

    VIO-11 retry/processor_service.py

    The cycle (soft):

    retry/processor_service.py
  └── [TYPE_CHECKING] imports BaseConsumer from consumers/base.py
      (annotates self._consumer: BaseConsumer)
      Uses: consumer.start(), consumer.stop(), consumer.is_running(), consumer._run_task

    Root cause: Over-specified annotation. BaseService already provides start(), stop(), is_running(), and _run_task. The concrete BaseConsumer reference is unnecessary — and creates an upward layer dependency from retry/ to consumers/.

    Fix:

    # Remove:
if TYPE_CHECKING:
    from ..consumers import BaseConsumer

# Change annotation:
from nala.athomic.services import BaseService

def __init__(self, consumer: BaseService) -> None:
    ...
    self._consumer: BaseService = consumer

    No new files needed. Remove from __future__ import annotations and TYPE_CHECKING import.

    Files to change: retry/processor_service.py

    Group 4: True Circular Imports — facade.py Abstraction Required

    VIO-13 plugins/interfaces.py

    The cycle:

    facade.py (Athomic)
  └── imports from plugins/ (plugin loading at startup)

plugins/interfaces.py
  └── [TYPE_CHECKING] imports Athomic from facade.py
      (used in: AthomicPlugin.setup(athomic: "Athomic") signature)

    Root cause: Plugins need to receive the facade at setup time, but they cannot import the concrete Athomic class without creating a cycle (facade → plugins → facade). The plugins need only a narrow interface from the facade — whatever methods they call during setup().

    Fix — extract AthomicCoreProtocol:

    New file: src/nala/athomic/plugins/core_protocol.py

    from typing import Any, Protocol


class AthomicCoreProtocol(Protocol):
    """Minimal interface that plugins receive during setup()."""

    @property
    def settings(self) -> Any: ...         # whatever plugins actually access

    def get_service(self, name: str) -> Any: ...  # narrow to actual usage
    # add only the methods plugins genuinely call — ISP applies

    Changes: - plugins/interfaces.py: remove TYPE_CHECKING guard around Athomic; use AthomicCoreProtocol as the parameter type in setup() and any other method. - facade.py (Athomic): must satisfy AthomicCoreProtocol structurally. - Audit which methods plugins actually call on athomic before writing the Protocol — ISP requires the Protocol to be minimal.

    Files to create: plugins/core_protocol.py Files to change: plugins/interfaces.py, optionally facade.py if the interface needs docs
