    Skew-Free Training-Set Generator — get_historical_features()

    Status: DELIVERED (commits 1–4) · Owner: features platform · Created: 2026-06-20

    Shipped 2026-06-20: training/ subpackage (HistoricalFeatureService, LineageAwarePointInTimeStoreProtocol, CatalogPlanHashResolver, TrainingPolicy, Arrow export) + FeaturePlatformService.get_historical_features. v1 was attest (Model A).

    Update 2026-06-22 — Model B + backfill shipped (§9). training/raw_state.py (RawStatePointInTimeStoreProtocol + MemoryRawStatePITStore) + training/recompute.py (HistoricalEncoderProtocol + RowRecomputer) + TrainingPolicy.RECOMPUTE_ON_MISMATCH (ROW_RECOMPUTED): stale/missing rows are re-derived from raw state through the current plan and admitted as skew-free; backfilling a newly-added feature is the same path. Falls back to Model A when no recomputer is injected. The versioning/aliases axis shipped separately (active-versioning + release consensus).

    1. Goal

    Add the defining capability of a state-of-the-art Feature Platform: generate a point-in-time-correct training set from a spine of (entity_id, timestamp) pairs — and do it with a property no other platform can offer: training/serving skew eliminated by construction, attested by plan_hash.

    result = await platform.get_historical_features(
    vector_id="customer_risk",
    spine=[("cust_1", t0), ("cust_2", t1), ...],   # entities × event times
)
result.matrix          # (N, dimension) feature matrix aligned to the spine
result.report          # coverage, skew attestation, version breakdown

    The caller joins its own labels to the spine externally; we deliver the skew-attested feature matrix aligned row-for-row to the spine.

    2. The original contribution (why this is publishable, not just parity)

    Tecton / Feast / Vertex generate training data through an offline engine (Spark / SQL) that is different from the online serving engine (Python / gRPC). The two code paths are kept "in sync" by convention, testing, and discipline — training/serving skew is an ever-present risk they mitigate, not eliminate.

    This platform compiles every feature vector to a single immutable FeatureExecutionPlan with a deterministic plan_hash (compiler/core.py:349). Materialized snapshots in the PIT store already carry the plan_hash that produced them (store/models.py:22).

    The claim: a training row is admitted to the dataset only when its materialization plan_hash equals the live serving plan_hash for that vector. Equal hashes ⇒ the bytes were produced by the identical compiled plan that serves online ⇒ skew-free by construction, not by convention. Rows that fail attestation are never silently mixed in — they are reported.

    Compiled-plan-attested point-in-time correctness: the same plan_hash gates both online serving and offline training-set admission, making training/serving skew structurally impossible rather than statistically unlikely.

    That is a defensible, original architectural contribution — it is only possible because of the AOT-compiled single-plan design.

    3. What exists vs what this adds

    Piece Status
    PIT vector retrieval get_as_of / get_batch_as_of ✅ store/protocol.py
    Per-snapshot lineage get_lineage_as_of / get_batch_lineage_as_of (carries plan_hash, vector_version) ✅ store/providers/lake.py
    Live serving plan_hash per vector ✅ FeatureCatalogEntry.plan_hash (catalog)
    Spine → PIT-join → skew-attested matrix orchestration ❌ this work
    Lineage in the PIT Protocol (ISP) ❌ this work (extend port)
    Coverage / skew / version report ❌ this work

    So this is pure orchestration + attestation over primitives that already exist. No new storage, no new engine.

    4. Honest scope — Model A (attest), not Model B (recompute)

    The PIT store holds already-computed feature vectors + lineage, not raw historical state. So v1 is retrieve-and-attest:

    • Pull PIT vectors for the spine via get_batch_as_of.
    • Pull their lineage via get_batch_lineage_as_of (the plan_hash each was computed under).
    • Attest each row against the current serving plan_hash.

    Out of scope for v1 (documented follow-up): recompute-on-mismatch (Model B) — re-running stale rows through the current plan — requires a raw-state PIT source (e.g. the streaming lake sink persisting raw events), which the store does not currently keep. v1 instead reports stale/missing rows rather than silently recomputing or dropping. Reporting skew instead of hiding it is itself a SOTA property most platforms lack.

    5. Components (SOLID / Hexagonal)

    5.1 Port extension — lineage in the PIT Protocol (ISP)

    get_batch_lineage_as_of lives only on the concrete lake provider today. Lift it into the port so the generator depends on an abstraction, not the adapter:

    @runtime_checkable
class LineageAwarePointInTimeStoreProtocol(PointInTimeStoreProtocol, Protocol):
    async def get_batch_lineage_as_of(
        self, requests: list[tuple[str, datetime]],
    ) -> dict[str, Optional[FeatureVersionInfo]]: ...

    (Separate Protocol so stores without lineage still satisfy the base port — ISP.)

    5.2 Plan-hash source — PlanHashResolverProtocol

    The generator needs the current serving plan_hash per vector. Don't couple it to the registry or service; inject a tiny port:

    class PlanHashResolverProtocol(Protocol):
    def current_plan_hash(self, vector_id: str) -> Optional[str]: ...

    Default impl reads FeatureCatalogEntry.plan_hash from the catalog (or encoder._plan.plan_hash from the registry). DIP: the generator depends on the port, composition root wires the concrete source.

    5.3 Orchestrator — HistoricalFeatureService (new training/ subpackage)

    Single responsibility: assemble a skew-attested training matrix. Offline / control-plane only — never on the serving hot path.

    class HistoricalFeatureService:
    def __init__(self, store: LineageAwarePointInTimeStoreProtocol,
                 plan_hashes: PlanHashResolverProtocol) -> None: ...

    async def get_historical_features(
        self, vector_id: str,
        spine: list[tuple[str, datetime]],
    ) -> TrainingSetResult: ...

    Algorithm: 1. current = plan_hashes.current_plan_hash(vector_id) (attestation reference). 2. vectors = await store.get_batch_as_of(spine); lineage = await store.get_batch_lineage_as_of(spine) (concurrent). 3. Per spine row, classify: - skew_free — vector present and lineage.plan_hash == current. - stale — vector present but plan_hash != current (materialized under an older plan; skew risk). - missing — no snapshot within the lookback window. 4. Assemble the matrix (skew-free rows; stale/missing → configurable policy: strict raises on any violation, attested_only emits skew-free rows + reports the rest, include_all emits everything tagged per-row). 5. Return TrainingSetResult.

    5.4 Result types

    @dataclass(frozen=True)
class TrainingRow:
    entity_id: str
    as_of: datetime
    values: Optional[list[float]]
    plan_hash: Optional[str]
    status: str   # "skew_free" | "stale" | "missing"

@dataclass(frozen=True)
class TrainingSetReport:
    vector_id: str
    current_plan_hash: str
    rows_total: int
    rows_skew_free: int
    rows_stale: int
    rows_missing: int
    coverage: float                       # skew_free / total
    stale_plan_hashes: dict[str, int]     # old_hash -> count (version drift map)

@dataclass(frozen=True)
class TrainingSetResult:
    matrix: FeatureTensor                 # domain type, never raw ndarray
    rows: list[TrainingRow]               # spine-aligned, row-for-row
    report: TrainingSetReport

    Matrix is built via FeatureTensorFactory.wrap(...) — the Protocol return-type rule (CLAUDE.md): the domain type crosses the boundary, never np.ndarray.

    5.5 Export (reuse existing io/)

    TrainingSetResult.to_arrow() via the existing ArrowAdapter for hand-off to pandas / Spark / Parquet. No new dependency.

    5.6 Facade wiring

    Expose on the platform as a control-plane operation, kept off the serving surface: platform.get_historical_features(vector_id, spine) delegating to the service. (When the Control/Serving plane split lands, this moves to the ControlPlane cleanly — it is already isolated here.)

    6. Failure modes & guards

    Risk Guard
    Silent training/serving skew Per-row plan_hash attestation; strict policy raises, others report rows_stale
    Label leakage (future data in past rows) Inherited from PIT store as_of semantics; generator never reads past as_of
    Vector not bootstrapped (no current hash) current_plan_hash returns None → raise ValueError (cannot attest)
    Store without lineage support isinstance(store, LineageAwarePointInTimeStoreProtocol) check at construction → clear error
    Huge spine get_batch_as_of is already a single-scan batch API; document a chunking helper for >1M rows
    Partial coverage hidden from caller report.coverage + per-row status always returned; never silently dropped

    7. Tests

    • Attestation matrix: spine with mixed lineage → assert skew_free / stale / missing classification exactly; stale_plan_hashes drift map correct.
    • Skew-free invariant (the headline test): materialize a batch via the live encoder, then get_historical_features over the same entities → every row status == "skew_free" and matrix equals the online encode_batch output byte-identically (the structural proof of the §2 claim).
    • Policy modes: strict raises on any stale/missing; attested_only emits skew-free + reports; include_all emits all tagged.
    • Port/DIP: a fake LineageAwarePointInTimeStoreProtocol + fake PlanHashResolverProtocol drive the service with no lake/registry.
    • Missing current hash → ValueError.
    • Arrow export round-trips matrix + row metadata.

    8. Commit shape

    • Commit 1: LineageAwarePointInTimeStoreProtocol (port) + make the lake provider satisfy it (it already has the method — just typed into the port); PlanHashResolverProtocol + catalog-backed default.
    • Commit 2: training/ subpackage: result types + HistoricalFeatureService (attestation algorithm) + the headline skew-free invariant test.
    • Commit 3: policy modes + report drift map + ValueError guards + DIP tests.
    • Commit 4: Arrow export via io/ArrowAdapter; facade wiring (platform.get_historical_features) + docs.

    Each behind the quality gate (mypy 0 · pyright 0 · pylint 10 · tests green).

    9. Sequenced follow-ups

    • Recompute-on-mismatch (Model B) — ✅ DELIVERED 2026-06-22. Raw-state PIT port + RowRecomputer + RECOMPUTE_ON_MISMATCH policy turn stale/missing rows skew-free by re-running the current plan. A production raw-state source (raw-event lake sink) is a pluggable adapter behind RawStatePointInTimeStoreProtocol (in-memory adapter ships).
    • Backfill — ✅ DELIVERED (same machinery): a newly-added feature over history starts all-missing and is filled by recompute.
    • Versioning/aliases/shadow/canary — the next SOTA axis; stale_plan_hashes here already produces the version-drift signal those features will act on.
    • Control/Serving plane split — this service is pre-isolated to move into the ControlPlane unchanged.
